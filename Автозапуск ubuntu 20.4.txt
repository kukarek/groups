Для настройки автозапуска Python файла на сервере Ubuntu 20.04 вы можете использовать различные методы, такие как создание systemd службы или добавление задачи cron. Вот как это сделать с использованием systemd:

Создайте файл для вашей службы в каталоге /etc/systemd/system/. Допустим, вы хотите запустить файл my_script.py.

  sudo nano /etc/systemd/system/my_script.service

Замените my_script на имя вашего файла без расширения.

Вставьте следующий блок текста в файл my_script.service, замените YourUsername на ваше имя пользователя, YourPythonScriptPath на полный путь к вашему Python скрипту, и YourVirtualenvPath на полный путь к виртуальной среде Python (если она используется):

  [Unit]
  Description=My Python Script Service
  After=network.target

  [Service]
  User=YourUsername
  ExecStart=/path/to/virtualenv/bin/python3   YourPythonScriptPath
  WorkingDirectory=/path/to/your/script/directory
  Restart=always

  [Install]
  WantedBy=multi-user.target

Обратите внимание, что User должен быть вашим именем пользователя, а ExecStart должен указывать на правильный путь к Python скрипту.

Сохраните и закройте файл.

Загрузите изменения в systemd:

  sudo systemctl daemon-reload

Запустите службу и включите автозапуск:

  sudo systemctl start my_script
  sudo systemctl enable my_script

Замените my_script на имя вашей службы.

Теперь ваш Python скрипт будет запускаться автоматически при загрузке сервера Ubuntu 20.04. Вы можете контролировать его с помощью команды systemctl.
